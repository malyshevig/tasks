# Резюме постановки задачи

## Цель
Создать одностраничное React-приложение (SPA) для управления задачами с использованием REST API.

## Основные требования

### 1. Получение данных
- Получать данные задач через GET-запрос к endpoint: `http://localhost:8080/api/tasks`
- Формат данных: JSON с полями:
  - `id` - идентификатор задачи
  - `name` - имя файла
  - `status` - статус задачи
  - `lines` - количество строк
  - `fail_count` - количество ошибок
  - `worker_id` - идентификатор воркера
  - `ts` - временная метка

### 2. Автоматическое обновление
- Обновлять данные каждые 9 секунд
- Предоставить кнопку для ручного обновления данных

### 3. Функциональность таблицы
- Отображать данные в табличном формате
- Реализовать фильтрацию по каждому столбцу
- Реализовать сортировку по каждому столбцу (по клику на заголовок)

### 4. Управление задачами
- Добавить столбец "Действия" с кнопками:
  - Для задач со статусом `in_progress` или `open`:
    - Кнопка "Pause"
    - POST-запрос на `http://localhost:8080/api/task/{id}/pause`
  - Для задач со статусом `paused`:
    - Кнопка "Resume"
    - POST-запрос на `http://localhost:8080/api/task/{id}/resume`
- После выполнения запроса обновлять данные задачи на основе ответа сервера

### 5. Пользовательский интерфейс
- Чистый, современный дизайн
- Адаптивная верстка
- Индикаторы загрузки
- Уведомления об операциях
- Цветовая индикация статусов задач

### 6. Обработка ошибок
- Отображение сообщений об ошибках подключения
- Обработка ошибок при выполнении действий

## Технические требования
- Использование React с функциональными компонентами и хуками
- Использование Bootstrap для стилизации
- Реализация без мок-данных, с реальными API-запросами

## Реализованное решение
Создано одностраничное React-приложение, удовлетворяющее всем требованиям, с:
- Автоматическим и ручным обновлением данных
- Фильтрацией и сортировкой по всем столбцам
- Кнопками управления задачами (Pause/Resume)
- Системой уведомлений
- Обработкой ошибок
- Современным пользовательским интерфейсом
